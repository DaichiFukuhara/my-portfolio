### 1. 変数
JavaScriptと同じかきかた
```ts
const x = 10;           // 基本これ
let y = 10;             // 再代入するときだけ
// var は使わない
```

型を明示した書き方
```ts
const name: string = "D";
const age: number = 22;
const isActive: boolean = true;
// const A: type = X;が基本の書き方
```

推論で足りるなら省略OK。**関数の引数と戻り値は明示**するのが原則。

### 3. オブジェクト型

3つの書き方があって、使い分けます。

#### `type`（エイリアス）


```ts
type User = {
  id: number;
  name: string;
  email?: string;       // ? で省略可
  readonly createdAt: Date;  // 読み取り専用
};

const u: User = { id: 1, name: "D", createdAt: new Date() };
```

#### `interface`


```ts
interface User {
  id: number;
  name: string;
}
```

**`type` と `interface` の違い**：

- `interface` は同じ名前で再定義してマージできる（拡張に強い）
- `type` は union や交差型など複雑な型を作れる
- 迷ったら `type`、ライブラリの公開APIなら `interface` が無難

### 4. Union と Literal

Web開発で一番使う型表現。
Literalは特定の値しか受け取らない。いうなれば特定の値を型とする。
```ts
let myName: "Fukuhara"; // myNameの型を「Fukuhara」とする

myName = "Fukuhara"; // OK
myName = "Tanaka"; // エラーとなる
```

Unionは「|」でつないだ複数の型を指定できる。
```ts
let Day: string | Date;

Day = "YYYY/MM/DD"; // OK
Day = new Date("2026/06/28"); // OK
Day = 2026; // エラーとなる
```

これらを組み合わせた使い方が頻繁に使われる。
```ts
type Status = "loading" | "success" | "error";   // 文字列リテラルUnion
let s: Status = "loading";

type ID = string | number;                       // どっちでもOK

type Theme = "light" | "dark";
function setTheme(t: Theme) { /* ... */ }
setTheme("dark");      // OK
setTheme("blue");      // ❌ コンパイルエラー
```

これだけで「マジックストリングのtypo」を全部潰せます。

### 5. 関数


```ts
// 関数宣言
//           引数　　　　　　　　　　　返り値の型
function add(a: number, b: number): number {
//　実行部
  return a + b;
}

// アロー、一行の関数のみを端的に表現する方法
//          引数　　　　　　　　　　　　返り値　　実行部
const add = (a: number, b: number): number => a + b;

// オプショナル引数、なくても動く引数
//            必須引数　　　　　オプショナル引数
function greet(name: string, title?: string): string {
  return title ? `${title} ${name}` : name;
}

// デフォルト引数、undefinedのとき設定されたものが代わりに使用される
//            必須引数　　　　　デフォルト引数
function greet(name: string, title: string = "Mr."): string {
  return `${title} ${name}`;
}
const S: string = greet("Fukuhara") // Mr.Fukuhara
const S: string = greet("Fukuhara", "Daichi") // DaichiFukuhara
const S: string = greet("Fukuhara", undefined) // Mr.Fukuhara

// 可変長引数、必要なメモリが呼び出し側で決定される
//           可変長引数
function sum(...nums: number[]): number {
//       reduceは配列を一つの値にするという関数reduce(アロー関数, 初期値)
  return nums.reduce((a, b) => a + b, 0);
}
sum(1, 2, 3);          // 6
sum(1, 2, 3, 4, 5);    // 15
sum();                 // 0
```

可変長関数は関数のとる引数で一番後ろに配置する
#### 関数型を変数に


```ts
type Handler = (event: MouseEvent) => void;
const onClick: Handler = (e) => console.log(e.clientX);
```

### 6. 配列・オブジェクト操作

#### スプレッド・分割代入


```ts
// 配列
const a = [1, 2, 3];
const b = [...a, 4];               // [1,2,3,4]
const [first, ...rest] = a;        // first=1, rest=[2,3]

// オブジェクト
const u = { id: 1, name: "D" };
const u2 = { ...u, age: 22 };      // 追加
const { id, name } = u;            // 分割代入
const { id: userId } = u;          // リネーム
```

#### よく使う配列メソッド


```ts
arr.map(x => x.id);                // 変換
arr.filter(x => x.active);         // 抽出
arr.find(x => x.id === 1);         // 1個見つける（無ければundefined）
arr.some(x => x.error);            // 1個でも条件満たす？
arr.every(x => x.valid);           // 全部条件満たす？
arr.reduce((acc, x) => acc + x.price, 0);  // 集約
```

### 7. Null / Undefined の扱い

TS の「`strictNullChecks`」が有効だと、`null` `undefined` は別の型として扱われます。Web開発で超重要。

ts

```ts
const el = document.getElementById("app");
// el の型は HTMLElement | null

el.textContent = "hi";   // ❌ null の可能性があるためエラー

// 解決パターン
if (el) {
  el.textContent = "hi";  // 中では HTMLElement になる
}

el?.textContent;          // optional chaining、null なら undefined を返す

el!.textContent = "hi";   // non-null assertion、「絶対nullじゃない」と断言（自己責任）

const text = el?.textContent ?? "default";   // null/undefined なら "default"
```

`?.` `??` `!` は Web TS で日常的に使います。

### 8. ジェネリクス

「型を引数として受け取る」仕組み。再利用性が一気に上がる。

ts

```ts
function identity<T>(x: T): T {
  return x;
}

identity<string>("hello");   // T = string
identity(42);                 // T = number（推論）

// 配列の最初を取る関数
function first<T>(arr: T[]): T | undefined {
  return arr[0];
}

first([1, 2, 3]);    // number | undefined
first(["a", "b"]);   // string | undefined
```

API呼び出しでよく使うやつ：

ts

```ts
async function fetchJson<T>(url: string): Promise<T> {
  const res = await fetch(url);
  return res.json();
}

const user = await fetchJson<User>("/api/user/1");
```

### 9. クラス

ts

```ts
class User {
  // フィールド宣言（型注釈）
  id: number;
  name: string;
  private password: string;       // クラス内のみ
  readonly createdAt: Date;       // 読み取り専用

  constructor(id: number, name: string, password: string) {
    this.id = id;
    this.name = name;
    this.password = password;
    this.createdAt = new Date();
  }

  greet(): string {
    return `Hi, ${this.name}`;
  }
}

const u = new User(1, "D", "secret");
```

#### 短縮形（パラメータプロパティ）

ts

```ts
class User {
  constructor(
    public id: number,
    public name: string,
    private password: string
  ) {}
}
```

これだけで `this.id = id` などが自動でされます。便利。

### 10. enum vs リテラルUnion

ts

```ts
// enum（昔ながら）
enum Color { Red, Green, Blue }
const c = Color.Red;   // 0

// リテラルUnion（推奨）
type Color = "red" | "green" | "blue";
const c: Color = "red";
```

最近は **リテラルUnion の方が好まれる**（バンドルサイズ・シンプルさ）。

### 11. Utility Types（神機能）

既存の型から新しい型を作るやつ。Web開発で多用します。

ts

```ts
type User = { id: number; name: string; email: string };

Partial<User>      // 全部オプショナルに：{ id?: number; name?: string; email?: string }
Required<User>     // 全部必須に
Readonly<User>     // 全部 readonly に
Pick<User, "id" | "name">      // 一部だけ：{ id: number; name: string }
Omit<User, "email">             // 一部除外：{ id: number; name: string }
Record<string, User>            // { [key: string]: User }
```

実例：

ts

```ts
function updateUser(id: number, patch: Partial<User>) {
  // patch は { name?: string; email?: string } など部分的でOK
}
```

### 12. 非同期処理（async/await）

ts

```ts
async function fetchUser(id: number): Promise<User> {
  const res = await fetch(`/api/users/${id}`);
  if (!res.ok) throw new Error("Failed");
  return res.json();
}

// 呼び出し
try {
  const user = await fetchUser(1);
  console.log(user.name);
} catch (e) {
  console.error(e);
}
```

`async` 関数の戻り値は必ず `Promise<T>`。

### 13. 型ガード

実行時に型を絞り込む。

ts

```ts
function isString(x: unknown): x is string {
  return typeof x === "string";
}

const val: unknown = getSomething();
if (isString(val)) {
  val.toUpperCase();   // ここでは string
}

// typeof / instanceof / in も型ガードになる
if (typeof x === "number") { /* x は number */ }
if (x instanceof Date) { /* x は Date */ }
if ("name" in obj) { /* obj に name プロパティがある */ }
```
